GENERIC MEMORY LIBRARY — DISK-DRIVEN TOPICS & BUNDLES
=====================================================

There is **no fixed list of categories** in code. Whatever you create under::

  backend/backend/static/memory/generic/

becomes the catalog after API restart.

Structure::

  static/memory/generic/<topic_folder>/<bundle_folder>/manifest.json
  static/memory/generic/<topic_folder>/<bundle_folder>/<any_image>.jpg

- **Topic** = first-level folder (e.g. ``cricket``). It appears in the app only if
  it contains at least one **bundle** subfolder that has ``manifest.json``.
- **Bundle** = subfolder with ``manifest.json`` + images. There is **no limit** on
  how many images you list in the manifest (each needs a ``title``).

Bundle ``manifest.json`` includes image keys (exact filenames) and a reserved
block for pricing::

  {
    "__bundle__": { "free": true },
    "photo1.jpg": { "title": "...", "location": "..." }
  }

Paid example::

  {
    "__bundle__": { "price_cents": 499, "currency": "USD" },
    "photo1.jpg": { "title": "..." }
  }

Omit ``__bundle__`` or omit price → bundle is treated as **free** (all patients
see it without purchase). Paid bundles require an unlock (caretaker purchase)
for patients.

Optional scripts (your data only — no random seeding):

  python scripts/export_generic_image_checklist.py
  python scripts/download_generic_images_from_manifest.py your_urls.json

Restart the API after changing folders or manifests so the DB sync runs.
